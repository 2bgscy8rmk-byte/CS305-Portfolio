CS 305 Portfolio – Artemis Financial

Client Summary

Artemis Financial is a financial services company that required improvements to the security of its software application. The organization needed to identify and mitigate vulnerabilities in order to protect sensitive financial and client data. My role was to perform a vulnerability assessment and evaluate the security posture of the application.

Identifying Vulnerabilities

I conducted a vulnerability assessment that included reviewing the application’s code and analyzing third-party dependencies. Static testing tools such as OWASP Dependency-Check were used to identify known vulnerabilities associated with external libraries. Secure coding is essential because it protects sensitive information, prevents exploitation, and ensures system reliability. Strong software security adds value to a company by protecting customer trust, reducing financial risk, and supporting regulatory compliance.

Challenges and Learning Experience

One challenging aspect of the vulnerability assessment was interpreting static testing results and distinguishing between true vulnerabilities and false positives. This process helped me better understand how automated security tools work and how to document risks appropriately.

Increasing Security Layers

To increase security layers, I reviewed dependency risks, recommended secure configuration practices, identified areas requiring stronger encryption, and documented mitigation strategies. In future projects, I would continue using OWASP tools, static analysis tools, and secure coding standards to assess vulnerabilities and determine appropriate mitigation techniques.

Ensuring Functionality and Security

After identifying vulnerabilities and recommending improvements, I ensured that security enhancements would not negatively impact system functionality. Security testing and documentation were used to verify that the application remained stable while reducing potential risks.

Professional Application

This artifact demonstrates my ability to analyze software for security vulnerabilities, document risks, apply mitigation strategies, and follow DevSecOps principles. It highlights my understanding of secure software development practices and my ability to contribute to a company’s overall security posture.
